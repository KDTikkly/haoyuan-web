## 一、产品愿景与目标

**产品定位**：Corealis 是 Lyria 的数字分身终端——一个高度个性化、兼具技术深度与视觉冲击力的作品集网站。它不是简历的替代品，而是一个**可以被感受**的空间。

**核心目标**：

1. **全面展示专业能力**：从 Web3/区块链探索、AI 驱动的虚拟世界项目（如 Cosmolyra），到工程实践（如 DMAIC 质量分析），系统性呈现多元化成果——这些都是真实做过的事，不是为了堆砌而堆砌。

2. **突显职业意向**：为求职视频内容运营与 AI 工作流整合相关岗位提供强有力的背景背书。如果你在找的那个人需要能拆解 AI 工作流、也能讲清楚为什么要这么做——那大概就是我。

3. **强化个人品牌**：Memphis 视觉 + ACG 文化背景 + 前卫 Web 交互，打造高辨识度的个人主页。我希望看到这个网站的人，第一反应不是"哦，又一个作品集"，而是——"这个人在认真玩"。

## 二、目标用户

- **招聘方/HR**：寻找具备 AI 工具整合能力、内容运营经验及技术背景的复合型人才。如果你打开这个网站，说明你愿意多花一点时间了解候选人，我也愿意用这个网站认真回应你的时间。

- **开源社区/项目协作者**：了解我的技术栈、项目管理经历及合作潜力。代码都在 GitHub 上，没什么可藏的。

- **ACG/游戏行业同行**：展示在动画俱乐部运营及游戏（如《重返未来：1999》）相关调研中的行业洞察。Steam 游戏库 127 款不是摆设，是有话要说。

## 三、当前系统架构与设计规范

### 3.1 技术架构（Vercel-Only Static Site）

- **核心框架**: Vue 3 (Composition API) + Vite
- **路由 & 状态**: Vue Router + @vueuse/core
- **数据层**: 纯静态读取 `public/data/projects.json` 与 `public/assets/`
- **国际化**: vue-i18n (中/英双语)

零后端，纯静态。不是因为懒，是因为没必要复杂——能用最简单的方式跑起来，就不加不必要的东西。

### 3.2 视觉语言（Memphis & Brutalist）

任何新需求的 UI 设计必须严格遵循以下规范：

- **色彩**: `warm-white` 底色，`memphis-yellow/blue/coral/mint` 作为高亮强调色
- **排版**: Brutalist 风格，使用 Inter/Space Grotesk 字体
- **形状与阴影**: 3px 纯黑实线边框 (`border-3 border-ink`) + 硬阴影 (`shadow-hard` 系列，无模糊)
- **动效**: GSAP 丝滑过渡 + matter-js 物理引擎背景交互

Memphis 的核心不是"好看"，是"有立场"。每一条硬边框都是刻意的选择，不是装饰。

## 六、非功能性需求

1. **性能 (Performance)**：所有新增图片必须经过压缩，Lighthouse 跑分保持在 90+。网站加载慢是一种失礼。

2. **响应式 (Responsive)**：移动端优先，所有 Memphis 元素在小屏幕下必须合理折叠，不出现横向滚动条。大多数人第一次打开都在手机上。

3. **SEO & 社交分享**：确保 `<meta>` 标签（Title, Description, Open Graph 图标为二次元头像）配置完整。如果有人愿意分享这个网站，至少要让它在预览卡片里好看。
